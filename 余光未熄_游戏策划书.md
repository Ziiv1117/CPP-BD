# 《余光未熄》游戏策划书

## 1. 项目概念

《余光未熄》是一款 2D 非对称光影对抗游戏。游戏核心围绕“光阵营三名人类协作完成净化任务”和“影阵营破坏灯源、制造黑暗、同化人类”展开。

光阵营由守灯人、工程师、观测员三名角色组成。三名角色不是数值不同的同类角色，而是承担不同职责：守灯人负责灯源和安全区，工程师负责机关和维修面板，观测员负责发现影的位置、残影和真实目标。影阵营则通过穿墙、传送、同化、破坏灯源、操控维修面板和制造分身来阻止任务推进。

最终目标是实现多人联机房间制：每名玩家只操控一名角色，通过分工、沟通和实时操作完成或阻止一局任务。

一句话概括：

```text
人类必须依靠三种职责点亮神殿，影则在黑暗中拆散协作，让余光彻底熄灭。
```

## 2. 核心设计原则

- 每名人类角色都有不可替代的任务职责。
- 所有人类都能点亮未点亮灯源，但只有守灯人能修复损坏灯源。
- 工程师是机关和路线推进核心，负责维修面板和地图状态变化。
- 观测员是信息核心，负责发现影的位置、影迹、残影和真实目标。
- 影不是正面战斗角色，而是破坏、欺骗、拖延和同化角色。
- 灯源是整局的核心资源，既是安全区，也是任务条件。
- 任务不是单人读条，而是由角色职责组合形成的协作流程。
- 第二版、第三版等版本只是通往最终成品的阶段，不代表最终完成。

## 3. 阵营设计

### 3.1 光阵营

光阵营由三名人类角色组成：

- 守灯人。
- 工程师。
- 观测员。

胜利条件：

- 完成一局任务链。
- 启动最终净化仪式。
- 在影同化全部人类或破坏关键封印前完成净化。

失败条件：

- 全部人类被同化。
- 大厅主灯长期损坏。
- 封印装置长期无法启动。
- 最终净化进度长期无法推进。

### 3.2 影阵营

影阵营由一名影角色组成。

胜利目标：

- 阻止人类完成任务。
- 破坏灯源和机关。
- 拆散人类队伍。
- 逐步同化全部人类。

影的核心玩法不是直接击杀，而是通过黑暗、错误信息、路线干扰和持续压力迫使人类失误。

## 4. 角色设计

### 4.1 守灯人

定位：

守灯人是光阵营的灯源维护角色，主要负责建立安全光区、修复被影破坏的灯源，并在关键时刻保护队友。

基础交互：

```text
E：点亮未点亮灯源
E：修复损坏灯源
E：参与任务
E：救援队友
```

主动技能：

```text
Q：强光眩晕
```

技能说明：

守灯人使用手电筒释放一次强光，使前方范围内的影短暂眩晕。

影被眩晕后：

- 短时间无法移动。
- 无法穿墙。
- 无法传送。
- 同化行为被打断。
- 影子分身被驱散。

职业特点：

- 点灯速度更快。
- 修灯速度更快。
- 唯一可以修复损坏灯源的人类角色。
- 是最终净化任务中恢复大厅主灯的关键角色。

示例数值：

```text
普通角色点灯任务：6 秒
守灯人点灯任务：4 秒
守灯人修复灯源：3 秒
强光眩晕持续：2 秒
强光眩晕冷却：15 秒
```

### 4.2 工程师

定位：

工程师是光阵营的机关处理角色，主要负责开门、操作维修面板、改变地图结构和推进机关任务。

基础交互：

```text
E：点亮未点亮灯源
E：开门
E：参与任务
E：救援队友
```

工程师可以点灯，但不能修复损坏灯源。

主动技能：

```text
Q：操作维修面板
```

技能说明：

工程师可以在维修面板处改变地图状态。

可操作内容包括：

- 打开机关门。
- 关闭机关门。
- 升起平台。
- 降下平台。
- 打开捷径。
- 封锁危险路线。
- 恢复部分机关通路。

职业特点：

- 机关任务完成速度更快。
- 维修面板操作速度更快。
- 开门和路线推进能力强。
- 是最终净化任务中启动封印装置的关键角色。

示例数值：

```text
普通角色机关任务：6 秒
工程师机关任务：4 秒
工程师操作维修面板：3 秒
```

### 4.3 观测员

定位：

观测员是光阵营的侦查预警角色，主要负责发现影的位置，并向队友提供危险提示。

基础交互：

```text
E：点亮未点亮灯源
E：参与任务
E：救援队友
```

观测员可以点灯，但不能修复损坏灯源。

主动技能：

```text
Q：影迹观测
```

技能说明：

观测员短时间显示影的位置，并把提示共享给队友。

显示内容包括：

- 影的大致位置。
- 影的移动方向。
- 影刚刚传送后的残影。
- 影子分身的异常提示。
- 附近危险区域。

职业特点：

- 提供影的位置提示。
- 帮助队伍判断安全路线。
- 降低影制造信息差的能力。
- 是最终净化任务中识别真实荒神核心的关键角色。

示例数值：

```text
显示持续时间：3 秒
技能冷却时间：20 秒
```

### 4.4 影

定位：

影是黑暗阵营角色，目标是阻止人类完成任务，并逐步同化全部人类。

核心玩法：

- 破坏灯源。
- 制造黑暗。
- 拆散人类队伍。
- 利用穿墙和传送干扰任务。
- 使用分身制造错误判断。
- 靠近人类积累同化值。

## 5. 影能力设计

### 5.1 穿墙

能力说明：

```text
影可以穿过普通墙体和隔断。
```

限制：

- 不能穿过强光覆盖区域。
- 被守灯人眩晕后无法穿墙。

### 5.2 传送

能力说明：

```text
影可以在黑暗区域或影缝之间传送。
```

限制：

- 不能在强光区域内传送。
- 不能传送到已点亮灯源附近。
- 被眩晕时不能传送。

### 5.3 同化

能力说明：

```text
影靠近人类后，会逐渐增加该人类的同化值。
```

同化值满后，人类失去行动能力或被判定失败。

反制方式：

- 队友救援。
- 进入灯源安全区。
- 守灯人强光眩晕影。
- 观测员提前发现影的位置。

### 5.4 破坏灯源

能力说明：

```text
影可以破坏已点亮的灯源。
```

破坏后：

- 灯源进入损坏状态。
- 损坏灯源无法被普通角色重新点亮。
- 只有守灯人可以修复。

限制：

- 破坏灯源需要短暂停留。
- 被手电照射会减速。
- 被强光眩晕会打断。
- 影子分身不能破坏灯源。

### 5.5 操控维修面板

能力说明：

```text
影可以操控部分维修面板，改变地图状态。
```

影可以：

- 关闭机关门。
- 打开危险路线。
- 封锁人类捷径。
- 改变平台位置。
- 干扰工程师操作。

### 5.6 影子分身

能力说明：

```text
影可以制造一个短时间存在的分身。
```

分身作用：

- 误导人类。
- 骗出守灯人的眩晕技能。
- 干扰观测员判断。
- 制造追击压力。

限制：

- 分身不能同化人类。
- 分身不能破坏灯源。
- 分身被强光照射后会消失。

## 6. 灯源规则

灯源共有三种状态：

```text
未点亮
已点亮
损坏
```

### 6.1 未点亮灯源

规则：

- 所有人类角色都可以点亮未点亮灯源。
- 点灯需要读条时间。
- 读条期间受到影干扰会被打断。

### 6.2 已点亮灯源

已点亮灯源会形成安全光区。

效果：

- 降低人类同化风险。
- 限制影传送。
- 暴露靠近的影。
- 为任务区域提供安全条件。

### 6.3 损坏灯源

规则：

- 影可以将已点亮灯源破坏为损坏状态。
- 损坏灯源无法被普通角色点亮。
- 只有守灯人可以修复损坏灯源。

## 7. 任务设计

一局任务按以下顺序推进：

```text
1. 重燃残灯
2. 维修机关门
3. 护光开门
4. 灯源校准
5. 影迹追踪
6. 最终净化
```

### 7.1 任务 1：重燃残灯

类型：

```text
守灯人单人任务
```

地点：

```text
中庭光井 / 旧钟楼 / 暗渠入口
```

任务目标：

守灯人需要恢复一个关键灯源，为队伍建立安全光区。

任务流程：

如果灯源是未点亮状态：

```text
守灯人靠近灯源
按 E 点亮灯源
读条完成后灯源恢复
```

如果灯源是损坏状态：

```text
守灯人靠近损坏灯源
按 E 修复灯源
修复完成后灯源自动点亮
```

成功效果：

- 该区域生成安全光区。
- 附近人类同化值缓慢下降。
- 影无法在该区域附近传送。
- 后续任务区域解锁。

影的干扰方式：

- 在守灯人读条时接近同化。
- 用分身骗出强光眩晕。
- 破坏其他灯源制造压力。
- 通过传送压迫守灯人路线。

### 7.2 任务 2：维修机关门

类型：

```text
工程师单人任务
```

地点：

```text
电闸房 / 裂影长廊入口 / 封印大厅前门
```

任务目标：

工程师需要操作维修面板，打开一扇关键机关门。

任务流程：

```text
工程师靠近维修面板
按 Q 操作维修面板
读条完成后机关门开启
```

成功效果：

- 打开新的通道。
- 缩短前往下一个任务点的路线。
- 让队伍避开黑暗区域。
- 解锁后续双人任务区域。

影的干扰方式：

- 操控另一个面板关闭辅助通道。
- 传送到工程师附近。
- 破坏附近灯源。
- 用分身制造假追击。

### 7.3 任务 3：护光开门

类型：

```text
双人任务
```

推荐组合：

```text
守灯人 + 工程师
```

地点：

```text
封印大厅前门
```

任务目标：

工程师操作维修面板打开封印门，守灯人维持旁边灯源，防止影破坏灯源和干扰工程师。

分工：

工程师：

```text
靠近维修面板
按 Q 操作面板
完成机关门开启
```

守灯人：

```text
点亮附近灯源
如果灯源被破坏，修复灯源
必要时用 Q 眩晕影
```

成功条件：

- 工程师完成维修面板操作。
- 任务期间至少有一个附近灯源保持可用。

中断条件：

- 工程师被影同化打断。
- 维修面板被影干扰。
- 附近灯源全部损坏且无人修复。

影的干扰方式：

- 破坏守灯人维护的灯源。
- 传送到工程师附近。
- 操控维修面板反向关门。
- 使用分身骗守灯人技能。

### 7.4 任务 4：灯源校准

类型：

```text
双人任务
```

推荐组合：

```text
守灯人 + 观测员
```

地点：

```text
镜光室
```

任务目标：

房间里有多个灯柱，其中只有部分是真正需要点亮的灯柱，其他是被影污染的假灯柱。观测员判断正确灯柱，守灯人负责快速点亮和修复。

分工：

观测员：

```text
按 Q 使用影迹观测
判断哪个灯柱被污染
给守灯人提示正确目标
```

守灯人：

```text
靠近正确灯柱
按 E 点亮灯柱
如果灯柱被影破坏，负责修复
必要时用 Q 眩晕影
```

成功条件：

- 点亮正确灯柱数量达到要求。
- 没有连续点亮错误灯柱。

失败惩罚：

- 点亮错误灯柱后，房间短暂熄灯。
- 影获得一次传送机会。
- 任务进度后退一部分。

影的干扰方式：

- 破坏已点亮灯柱。
- 制造影子分身干扰判断。
- 趁观测员技能冷却时接近。
- 诱导守灯人点错灯柱。

### 7.5 任务 5：影迹追踪

类型：

```text
双人任务
```

推荐组合：

```text
工程师 + 观测员
```

地点：

```text
裂影长廊 / 暗渠 / 荒神核心外围
```

任务目标：

观测员找出影留下的正确痕迹方向，工程师操作维修面板打开正确路线。

分工：

观测员：

```text
按 Q 显示影的位置或影迹方向
判断哪条路线更安全
提示工程师操作哪个面板
```

工程师：

```text
靠近维修面板
按 Q 操作面板
打开正确通道
关闭危险通道
```

成功条件：

- 工程师打开正确路线。
- 队伍进入荒神核心外围。

失败惩罚：

- 打开错误通道后，队伍进入黑暗区域。
- 影获得额外传送点。
- 任务进度延迟。
- 附近灯源更容易被破坏。

影的干扰方式：

- 用分身制造假影迹。
- 操控维修面板改变门的状态。
- 破坏附近灯源制造黑暗。
- 传送到错误路线附近诱导人类判断。

### 7.6 任务 6：最终净化

类型：

```text
三人任务
```

参与角色：

```text
守灯人
工程师
观测员
```

地点：

```text
封印大厅
```

任务目标：

三名人类必须完成各自职责，启动最终净化仪式，封印荒神核心。

第一阶段：恢复大厅主灯

守灯人：

```text
靠近大厅主灯
按 E 修复灯源
修复完成后大厅主灯点亮
```

第二阶段：观测员定位真实核心

观测员：

```text
站在观测阵位
按 Q 使用影迹观测
识别真正的荒神核心
排除影子分身制造的假核心
```

如果没有完成该步骤：

- 工程师无法启动正确封印装置。
- 最终净化无法进入下一阶段。

第三阶段：工程师启动封印装置

工程师：

```text
靠近维修面板
按 Q 操作封印装置
连接最终净化线路
```

如果工程师被打断：

- 封印装置停止。
- 需要重新操作。

第四阶段：三人站位净化

三名人类分别站在对应阵位：

```text
守灯人：光源阵位
工程师：机关阵位
观测员：观测阵位
```

当满足以下条件时：

```text
大厅主灯处于点亮状态
真实核心已被观测员识别
封印装置已被工程师启动
三人都站在对应阵位
```

净化进度开始增长。

胜利条件：

```text
净化进度达到 100% 后，人类胜利。
```

影的干扰方式：

- 破坏大厅主灯。
- 操控维修面板打断工程师。
- 使用分身制造假核心。
- 传送到黑暗角落袭击观测员。
- 同化守灯人，阻止其修灯。

人类反制方式：

- 守灯人修复灯源并眩晕影。
- 工程师重新操作面板。
- 观测员提示影的位置。
- 队友互相救援。
- 三人维持站位。

失败条件：

- 全部人类被同化。
- 大厅主灯长时间损坏。
- 封印装置长期无法启动。
- 最终净化进度长期无法推进。

## 8. 一局任务流程

完整一局推荐流程：

```text
1. 守灯人完成“重燃残灯”
2. 工程师完成“维修机关门”
3. 守灯人 + 工程师完成“护光开门”
4. 守灯人 + 观测员完成“灯源校准”
5. 工程师 + 观测员完成“影迹追踪”
6. 三名人类共同完成“最终净化”
```

## 9. 地图设计

地图采用横版隔断式结构，重点服务角色分工和任务路线。

基础区域：

```text
[旧钟楼] —— [上层廊桥] —— [镜光室]
   |             |             |
[电闸房] —— [中庭光井] —— [封印大厅]
   |             |             |
[暗渠]   —— [裂影长廊] —— [荒神核心外围]
```

区域职责：

- 中庭光井：重燃残灯、安全区建立。
- 旧钟楼：早期灯源任务和全局提示。
- 暗渠入口：低光、高风险路线。

## 11. 联机设计

最终形态是多人联机房间制。

核心绑定：

```text
PlayerId -> CharacterId
```

规则：

- 每名玩家只操控一名角色。
- 守灯人、工程师、观测员分别由不同玩家操控。
- 影由单独玩家操控。
- 当前版本中的本地切换角色只用于测试，不代表最终玩法。

房间流程：

```text
创建房间
玩家加入
选择或分配角色
准备
进入对局
完成或失败
结算
```

## 12. 音画风格

整体风格：

- 暗色神殿。
- 暖色灯源。
- 冷色阴影。
- 光照区域和黑暗区域对比强烈。
- 影的表现应具有雾化、破碎和不稳定感。

角色表现：

- 守灯人：灯具、护光、暖色符号。
- 工程师：机关工具、面板、金属感。
- 观测员：镜片、观测仪、蓝绿色提示。
- 影：黑雾、紫色边缘、残影和分身。

音效重点：

- 点灯。
- 修灯。
- 操作面板。
- 影传送。
- 影破坏灯源。
- 强光眩晕。
- 影迹观测。
- 任务完成。
- 最终净化。

## 13. 版本路线

版本划分只是为了逐步实现最终版，不表示中间版本是成品。

### 13.1 版本 0：基础原型

验证窗口、地图、角色移动、基础光照和胜负判断。

### 13.2 版本 1：基础对抗版

验证人类手电、影穿墙、传送、同化和基础任务。

### 13.3 版本 2：阶段演示版

完成可打开、可展示、可操作、可跑通一局的阶段版本。

重点是验证入口、UI、移动、灯源、角色职责和基础任务闭环。

### 13.4 版本 3：素材导入版

导入角色、地图、灯源、UI、音效和特效素材，使画面更接近最终表现。

### 13.5 版本 4：完整任务链版

完整实现六个任务：重燃残灯、维修机关门、护光开门、灯源校准、影迹追踪、最终净化。

### 13.6 版本 5：联机架构版

实现房间、玩家槽位、角色绑定和基础同步。

### 13.7 版本 6：玩法扩展版

扩展动态地图、更多影干扰方式、更多灯源状态和更完整结算。

### 13.8 终版

形成多人联机、三人类职业协作、影干扰、完整任务链、正式素材和完整 UI 的最终可交付游戏。

## 14. 设计逻辑总结

```text
全部人类都可以点灯，但点灯需要时间。
影可以破坏已点亮灯源。
灯源被破坏后进入损坏状态。
损坏灯源只有守灯人可以修复。
工程师负责机关和地图变化。
观测员负责显示影的位置和提示危险。
影通过破坏灯源、穿墙、传送、同化、维修面板和分身来拆散人类。
最终任务要求三名人类分别承担光源、机关、观测三个职责。
```
