# 《余光未熄》第二版阶段推进书

## 1. 文档定位

本文档用于推进《余光未熄》的第二版开发。

第二版不是成品，也不是最终完成版。它只是通往最终版过程中的一个阶段版本，用来把当前项目从基础原型推进到可运行、可展示、可继续扩展的状态。

版本划分的意义是：

```text
把终版拆成多个可实现阶段。
每个版本解决一部分问题。
第二版只是中间阶段，不代表最终完成。
```

最终目标仍然是：

```text
多人联机房间制。
守灯人、工程师、观测员、影分别由不同玩家操控。
每名玩家只操控一名角色。
```

## 2. 第二版阶段目标

第二版的目标不是实现完整终版，而是完成一个能支撑后续开发的阶段演示版。

第二版目标：

- 有可运行的 Raylib 图形化游戏。
- 有可网页打开的 WebAssembly 版本。
- 有完整入口流程。
- 有基本 UI。
- 有角色移动和基础光照。
- 有人类与影的基础对抗。
- 有一条能跑通的任务链。
- 有胜负结算。
- 有后续改成正式角色分工和联机的基础。

第二版不要求完成：

- 真正网络联机。
- 终版全部 6 个任务的完整细节实现。
- 完整守灯人 / 工程师 / 观测员专属数值系统。
- 完整影分身、面板干扰和复杂灯源破坏读条系统。
- 完整正式素材。
- 完整音效和音乐。
- 公网部署。

当前仍属于阶段功能样机，不应被描述为成品。

## 4. 新设计对第二版的影响

根据最新设计，终版角色和任务应调整为：

人类角色：

- 守灯人。
- 工程师。
- 观测员。

影角色：

- 影。

终版任务链：

```text
1. 重燃残灯
2. 维修机关门
3. 护光开门
4. 灯源校准
5. 影迹追踪
6. 最终净化
```

因此，第二版不再继续沿用泛泛的“三灯校准、双闸供能、运送光核”路线，当前基础任务链已经迁移到新设计，后续重点是补齐每个任务的完整细节。

## 5. 第二版应保留的范围

第二版仍然只做阶段目标，不直接吃下终版全部系统。

第二版基础版已经完成：

- 把三名人类命名和定位改为守灯人、工程师、观测员。
- UI 中显示三名角色的职责。
- 让 E 成为基础交互键。
- 让 Q 成为角色主动技能键。
- 实现灯源三状态：未点亮、已点亮、损坏。
- 实现“损坏灯源只有守灯人能修复”。
- 实现守灯人强光眩晕的基础版本。
- 实现工程师维修面板的基础版本。
- 实现观测员影迹观测的基础版本。
- 将现有任务文案改为新任务链。
- 新六任务链均有基础判定。
- 最终净化已有简化版流程，文档和 UI 按三人职责设计。

第二版暂不强求：

- 完整影子分身。
- 完整假核心判断。
- 完整假灯柱惩罚。
- 完整影操控维修面板。
- 完整 6 任务全部细节。
- 真联机。

## 6. 当前完成与待修改事务

### 6.1 本次已落地内容

本次已经把第二版从旧任务样机推进到新设计的基础可跑通版本。

已完成：

- 三名人类改为守灯人、工程师、观测员。
- 本地演示保留切换角色能力，但按终版职业职责显示。
- 控制方式改为 `Tab/C` 切换人类、`E` 基础交互、`Q` 职业技能。
- 所有人类可以点亮未点亮灯源。
- 守灯人可以修复损坏灯源。
- 守灯人可以用强光眩晕影。
- 工程师可以通过维修面板推进机关门、路线和封印装置。
- 观测员可以用影迹观测暴露影位置，并推进灯源校准、影迹追踪、真实核心识别。
- 影可以移动、穿墙、传送、同化，并可以破坏已点亮灯源。
- 影被眩晕后无法移动、传送和同化。
- UI 任务链替换为新六任务。
- 新六任务基础流程可以从第一任务跑到最终净化。

### 6.2 自测结果

已执行验证：

- 桌面版构建通过。
- 桌面版普通启动烟测通过。
- 第二版任务链自动自测通过。
- WebAssembly 网页版构建通过。
- 浏览器打开 `http://localhost:8080` 后主菜单、准备页、游戏内 UI 均可显示。

自测中发现并已修复：

- 最终净化和护光开门曾共用同一维修面板坐标，导致工程师最终阶段按 `Q` 仍触发护光开门逻辑。已调整判定顺序，最终净化流程可通过。

### 6.3 待修改事务

这些不是第二版基础跑通的阻塞项，但会影响后续版本质量：

- 当前六任务是基础判定版，还没有读条、多人同步确认、中断进度和失败惩罚的完整细节。
- 工程师维修面板目前主要改变任务状态，还没有真实机关门、平台、捷径和危险路线的地图碰撞变化。
- 观测员影迹观测目前主要显示影所在房间和推进任务，还没有移动方向、残影、分身异常和危险区域的完整提示。
- 影可以破坏灯源，但还没有破坏读条、冷却、被手电减速和被强光中断的完整表现。
- 影子分身尚未实现。
- 影操控维修面板尚未实现。
- 灯源校准的真假灯柱惩罚还没有完整做出。
- 最终净化现在是简化版，还没有严格的光源阵位、机关阵位、观测阵位三人站位和 100% 净化进度条。
- 右侧 UI 已能显示新任务，但任务文本仍偏密，后续需要继续做排版和图标化。
- 本地演示仍是一台电脑操控多角色，终版需要改为联机下每名玩家只操控一名角色。

## 7. 第二版阶段任务拆分

### 阶段 A：文档和 UI 对齐

目标：

让项目文字全部按新设计表达。

任务：

- README 文档入口改名。
- 游戏策划书按新设计更新。
- 推进书改名为第二版阶段推进书。
- UI 文案明确第二版不是成品。
- 操作说明改为 E 基础交互、Q 主动技能。

验收：

- 文档不再把第二版描述成最终成品。
- 第二版定位明确为阶段版本。

### 阶段 B：角色职责落地

目标：

把三名人类改成守灯人、工程师、观测员。

任务：

- UI 名称替换。
- 角色说明补充。
- 守灯人 Q：强光眩晕基础版。
- 工程师 Q：维修面板基础版。
- 观测员 Q：影迹观测基础版。

验收：

- 三名角色在玩法上开始出现差异。
- 玩家能理解每个角色要做什么。

### 阶段 C：灯源三状态

目标：

实现灯源从单纯点亮变成可争夺资源。

任务：

- 灯源状态：未点亮、已点亮、损坏。
- 所有人类可点亮未点亮灯源。
- 影可破坏已点亮灯源。
- 守灯人可修复损坏灯源。
- 损坏灯源有视觉提示。

验收：

- 影能制造灯源压力。
- 守灯人成为不可替代角色。

### 阶段 D：维修面板和机关门

目标：

让工程师成为路线推进角色。

任务：

- 新增维修面板。
- 新增机关门状态。
- 工程师操作面板打开通道。
- 影可干扰或反向操作面板的基础版本。

验收：

- 工程师可以通过面板改变地图通路。

### 阶段 E：任务链迁移

目标：

将第二版任务从旧任务链迁移到新任务链，并先保证基础流程可以跑通。

当前基础实现：

1. 重燃残灯。
2. 维修机关门。
3. 护光开门。

后续需要完善：

- 每个任务的读条、中断、失败惩罚和提示。
- 任务区域的真实地图对象和碰撞变化。
- 最终净化的三人站位和进度条。

验收：

- 6 个任务的基础流程能跑通。
- UI 显示的是新任务名。

## 8. 第二版验收标准

第二版阶段版本可以验收的标准：

- 能双击打开网页。
- 能进入游戏。
- 能看到守灯人、工程师、观测员、影的角色定位。
- 能进行基础移动和光照。
- 能使用 E 进行基础交互。
- 能使用 Q 触发至少部分角色技能。
- 灯源至少具备未点亮、已点亮、损坏的基础表现。
- 至少一部分新任务链能跑通。
- 结算能显示胜负。
- 文档明确第二版不是成品，而是通往终版的阶段。

## 10. 当前结论

第二版不是成品。第二版只是为了逐步实现终版而设置的阶段版本。

当前项目已经有可运行基础，但后续应该按新设计逐步重构：

```text
先统一文档和 UI。
再落地三角色职责。
再补灯源三状态。
再补维修面板和机关门。
再迁移任务链。
最后再做素材、联机和完整终版。
```
